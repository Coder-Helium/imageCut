# 端到端图像构图裁剪模型数据集构建方案

## 1. 目标

本方案用于构建一个面向端到端图像构图裁剪模型的数据集。

核心目标不是简单生成一个 `best_crop`，而是为每张图构建完整的构图监督信息：

```text
图像语义理解
+ 主体/关键物体/背景 mask
+ 当前构图状态
+ 方向性裁剪动作
+ 多候选裁剪框
+ 每个候选框的质量分数和排序
+ 最终 best_crop
```

最终数据集可以支持三类训练任务：

```text
1. Crop ranker:
   输入 image + candidate crop，预测该 crop 的质量分数。

2. End-to-end crop generator:
   输入 image，直接输出 top-k crop boxes + scores。

3. Direction-aware composition model:
   输入 image，预测当前构图问题、调整方向和目标 crop。
```

## 2. 总体思想

本方案结合三类思路：

### 2.1 InstantRetouch 式数据构造

InstantRetouch 的数据构造思想是：

```text
高质量目标图
  -> 人为制造低质量输入图
  -> 生成 before-after 训练对
  -> 训练模型恢复目标效果
```

迁移到裁剪任务中：

```text
高质量构图 crop*
  -> 人为制造坏构图 crop_bad / loose image
  -> 生成构图问题和调整方向标签
  -> 训练模型从坏构图调整到好构图
```

例如：

```text
主体偏左        -> move_crop_left / place_subject_center
脚部太贴边      -> move_down_or_zoom_out
关键物被裁掉    -> preserve_relation
背景干扰物过多  -> zoom_in / remove_distractor
主体太满        -> zoom_out
主体太小        -> zoom_in
```

### 2.2 GAIC 式候选框排序

GAIC 论文的关键启发是：

```text
裁剪不是预测唯一正确框，而是在多个合理候选框中评分和排序。
```

因此，本数据集不只保存一个 `best_crop`，而是保存：

```text
candidate_boxes
+ candidate_scores
+ candidate_ranks
+ candidate_actions
+ candidate_reasons
```

这样可以训练 crop ranker，并且避免单一框监督导致的构图多解问题。

### 2.3 结构感知与方向性裁剪

裁剪需要知道：

```text
主体是谁
主体在哪里
主体是否完整
主体应该往哪边移动
主体和关键物体的关系是否要保留
背景是重要环境还是可裁掉区域
边缘是否有干扰物
```

所以数据构建阶段引入：

```text
VLM / caption 模型：理解主体、关系、场景和构图问题
Grounding / YOLO：定位主体和关键物体
SAM / 开放词表分割：生成精确 mask
规则系统：生成方向性候选 crop
美学模型 / VLM / 规则分数：给候选 crop 打分
```

这些模块主要作为离线 teacher 使用。最终端到端模型推理时不一定需要继续调用它们。

## 4. 数据输入

### 4.1 必需输入

每张原始图片：

```text
image_path
image_width
image_height
```

支持格式：

```text
jpg / jpeg / png / webp
```

### 4.2 可选输入

如果已有 caption 或语义分类结果，可以作为输入加速构建：

```json
{
  "image_path": "images/000001.jpg",
  "caption": "A woman holding a camera near a stone railing.",
  "semantic_type": "person_holding_object"
}
```

如果没有 caption，则由 VLM 自动生成结构化理解结果。

### 4.3 目标比例输入

裁剪任务通常需要支持不同目标比例：

```json
{
  "target_aspect_ratios": [
    "original",
    "1:1",
    "4:5",
    "3:4",
    "16:9",
    "9:16"
  ]
}
```

每个目标比例可以单独生成一组候选 crop 和 `best_crop`。

## 5. 数据输出

每张图片最终输出一个 JSONL 样本。一个样本对应一张图片和一个目标比例。

推荐文件：

```text
dataset/
  images/
  masks/
  visualizations/
  metadata/
    train.jsonl
    val.jsonl
    test.jsonl
    failed.jsonl
```

每行 JSONL 表示一个训练样本：

```json
{
  "sample_id": "000001__4x5",
  "image_path": "images/000001.jpg",
```

## 7. 选择性定位与分割

### 7.1 为什么不分割全图

数据构建阶段可以使用重模型，但仍不建议无差别分割全图所有区域。

原因：

```text
1. mask 粒度不稳定，有时整体人，有时头发/衣服/手分开。
2. 类别标签不稳定，同一物体可能被称为不同名字。
3. 大量无关 mask 会增加噪声。
4. 裁剪真正需要的是关键区域，不是所有区域。
```

因此，只分割裁剪相关区域：

```text
主体
关键物体
重要背景
干扰物
```

### 7.2 推荐流程

```text
VLM 输出需要保留/处理的对象文本
  |
Grounding-DINO / YOLO / YOLO-World / VLM bbox 定位
  |
SAM 使用 bbox prompt 生成精确 mask
  |
mask 质量检查
  |
保存 mask metadata
```

## 8. Crop-State Graph

### 8.1 目的

普通 scene graph 关注语义关系，例如：

```text
person holds camera
person stands near railing
```

裁剪更需要 crop-state graph，关注：

```text
主体是否完整
主体是否贴边
主体位置是否合理
关键物体是否被保留
背景是否重要
干扰物是否应该裁掉
```

## 10. 候选 Crop 生成

每张图不要只生成一个候选框。推荐混合五类来源：

### 10.1 GAIC Grid Anchor 候选

使用网格 anchor 枚举基础候选，提供全局覆盖。

约束：

```text
候选框在图内
满足目标宽高比
面积不小于原图一定比例
可覆盖常见构图位置
```

作用：

```text
保证候选空间完整，避免规则漏掉好 crop。
```

### 10.2 语义规则候选

来自现有 `caption-rule-co` 中的规则系统：

```text
single_static_portrait
single_action_portrait
double_relation
group_portrait
person_holding_object
person_animal
person_vehicle
person_nature
person_architecture
landscape_scene
detail_scene
```

作用：

```text
复用已有人工策略经验，生成高质量 teacher candidates。
```

### 10.3 Mask 保护候选

根据 `preserve_box`、`relation_box`、`important_region_box` 生成候选：

```text
完整包含主体
完整包含关键物
完整包含主体和关键物关系区域
为主体留出 padding
避免裁切头、手、脚、道具
```

### 10.4 方向性候选

根据 crop-state graph 中的 issue/action 生成：

```text
move_left
move_right
move_up
move_down
zoom_in
zoom_out
place_subject_left_third
place_subject_right_third
remove_distractor
keep_environment
```

例如：

```text
issue = subject_bottom_too_tight
候选：
1. crop 向下移动
2. crop 放大以包含更多下方区域
3. 保持主体完整并增加底部 padding
```

### 10.5 负样本候选

需要主动生成坏 crop，帮助 ranker 学会区分错误裁剪。

负样本类型：

```text
裁掉头
裁掉脚
裁掉手
裁掉关键物体
主体过小
主体过满
主体严重偏边
只保留背景
裁掉重要环境
保留大量干扰物
```

## 12. 候选框评分

### 12.1 总分建议

每个候选框的总分可以由多项组成：

```text
final_score =
  w1 * subject_integrity
+ w2 * key_object_integrity
+ w3 * relation_preservation
+ w4 * composition_position
+ w5 * environment_preservation
+ w6 * distractor_removal
+ w7 * aesthetic_score
+ w8 * vlm_preference
- penalty
```

一个初始权重示例：

```text
subject_integrity       0.25
key_object_integrity    0.15
relation_preservation   0.15
composition_position    0.15
environment_preservation 0.05
distractor_removal      0.10
aesthetic_score         0.10
vlm_preference          0.05
```

可以按语义类型动态调整。

例如，人像类：

```text
subject_integrity 权重大
composition_position 权重大
environment_preservation 权重中等
```

人与物体类：

```text
relation_preservation 权重大
key_object_integrity 权重大
```

风景类：

```text
environment_preservation 权重大
aesthetic_score 权重大
subject_integrity 权重低
```

### 12.2 主体完整性分数

基于主体 mask 计算：

```text
subject_coverage = crop 内主体 mask 面积 / 全主体 mask 面积
```

建议：

```text
coverage >= 0.98 -> 5.0
coverage >= 0.95 -> 4.5
coverage >= 0.90 -> 3.5
coverage >= 0.80 -> 2.0
otherwise        -> 1.0
```

如果裁掉头、脸、手、脚等关键部位，可以额外强扣分。

### 12.3 关键物体完整性分数

对 key object mask 计算 coverage：

```text
key_object_coverage = crop 内关键物 mask 面积 / 关键物 mask 面积
```

如果关系物体重要性较高，例如：

```text
person holding camera
person with dog
person riding bicycle
```

则 key object coverage 不足时应强扣分。

### 12.4 关系保留分数

关系保留不只看两个对象是否分别完整，还要看它们之间的空间关系。

可计算：

```text
relation_box_coverage
subject-key_object distance preserved
union_box contained
```

示例：

```text
人和相机都完整，但相机被裁到边缘很紧 -> 中等分
人完整，相机被裁掉 -> 低分
人和相机完整，关系区域舒适 -> 高分
```

### 12.5 构图位置分数

根据语义类型和动作计算：

```text
单人人像：主体中心或三分线位置
动作人像：动作方向留白
人物朝向：视线方向留白
群体照：群体 union 居中
风景：地平线、天空/地面比例
建筑：垂直线、对称性
```

### 12.6 干扰物去除分数

基于 distractor mask：

```text
distractor_removed_ratio = 被裁掉的干扰物面积 / 干扰物总面积
```

注意：

```text
如果干扰物和主体重叠，不应强行裁掉。
如果干扰物位于边缘，裁掉优先级更高。
```

### 12.7 VLM 偏好评分

可以让 VLM 对 top candidates 做 pairwise 或 listwise 评分：

```text
给定原图、候选裁剪图 A/B/C/D，选择构图最自然、主体最完整、关系最清楚的一张。
```

VLM 评分更适合用于：

```text
候选框数量已经较少时
top-k 复核
过滤规则分数高但视觉不自然的样本
```

## 15. 质量过滤

### 15.1 样本级过滤

建议过滤以下样本：

```text
主体定位失败
主体 mask 明显错误
候选框数量过少
所有候选框分数都很低
top1 和 top2 分数差距过小但视觉差异很大
VLM 理解和 mask 结果冲突
图片本身分辨率过低
图片含严重水印/拼接/异常比例
```

### 15.2 候选级过滤

过滤以下候选：

```text
框超出图像边界
宽高比不符合目标比例
面积过小
主体 coverage 过低
关键物体 coverage 过低
主体贴边过紧
裁剪后分辨率过低
```

### 15.3 人工复核优先级

优先人工复核：

```text
高分但裁掉主体的候选
低分但看起来合理的候选
top1/top2 差距极小的样本
复杂多人/多主体图
人与物体关系复杂图
VLM 和规则判断冲突图
```

## 16. 训练任务设计

### 16.1 Stage 1: Crop Ranker

输入：

```text
image
candidate_box
optional semantic_type
optional mask features
```

输出：

```text
candidate_score
```

监督：

```text
final_score
rank
pairwise preference
```

推荐 loss：

```text
Huber / MSE score regression
Pairwise ranking loss
Listwise ranking loss
```

### 16.2 Stage 2: End-to-End Crop Generator

输入：

```text
image
target_aspect_ratio
optional semantic_type
```

输出：

```text
top-k crop boxes
top-k crop scores
```

监督：

```text
best_crop
top-k high quality candidates
candidate rank distribution
```

推荐结构：

```text
Backbone: ConvNeXt / Swin / ViT
Decoder: DETR-style crop queries 或 grid-anchor prediction head
Output: K boxes + K scores
```

### 16.3 Stage 3: Direction-Aware Multi-task Training

额外输出：

```text
main_issue
best_action
semantic_type
```

监督：

```text
issue labels
action labels
reason text 可选
```

作用：

```text
让模型不仅知道裁到哪里，还知道为什么这样裁。
```

### 16.4 Stage 4: 人工/VLM 偏好微调

收集：

```text
人工 A/B 选择
用户点击/选择行为
VLM pairwise preference
```

用于：

```text
修正规则分数偏差
提高主观审美一致性
适配产品风格
```

当前项目已有：

## 18. 最小可行版本

如果先做 MVP，不需要一次实现所有模块。

推荐 MVP：

```text
1. 使用已有 caption / semantic_router。
2. 用 YOLO 找 person / object bbox。
3. 用 SAM 只分割主体和关键物。
4. 用现有规则 + grid anchor 生成候选。
5. 用 mask coverage + TorchEAT + 简单构图规则打分。
6. 保存 JSONL candidates + scores + best_crop。
```

## 19. 关键原则

```text
1. 数据构建阶段可以重，模型推理阶段再轻量化。
2. 不要只保存 best_crop，要保存 candidates + scores。
3. 不要分割全图所有东西，只分割裁剪相关区域。
4. 不要让 SAM 决定主体，SAM 只负责精细边界。
5. 主体、关系物和重要背景要有不同 importance。
6. 裁剪标签要包含方向性 action，而不是只有 box。
7. 分数要由多维规则组成，不能只依赖美学模型。
8. VLM 适合做语义理解和 top-k 复核，不适合完全替代几何规则。
9. 负样本必须主动构造，否则 ranker 学不到错误裁剪。
10. 最终端到端模型学习的是离线 teacher 的综合判断。
```

## 20. 一句话总结

本方案的核心是：

```text
用 VLM/SAM/规则/美学模型在离线阶段构建高质量 teacher 数据，
把每张图变成“语义结构 + 关键 mask + 构图问题 + 方向性候选 + 多维评分排序”，
再训练一个端到端模型学习从图像到 crop boxes、scores 和裁剪动作的映射。
```
